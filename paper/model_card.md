# Model card — Latent Human Dynamics Engine v0.1

## Model details

**Architecture.** A six-dimensional latent state-space model with three pieces. An encoder maps daily wearable, behavioral, environmental and missingness streams to a latent state $z_t \in \mathbb{R}^6$. A forced contractive transition model $z_{t+1} = (1-\kappa) z_t + \alpha F_{\text{env}}(E_t) + \alpha F_{\text{beh}}(B_t) + P_t + \epsilon_t$ steps the state forward. A counterfactual perturbation operator with seven primitives returns an additive latent delta over a configurable horizon. A classical PCA-based encoder is provided as a deterministic CPU fallback when no trained neural checkpoint is available.

**Latent dimensions.** `autonomic_recovery`, `circadian_alignment`, `stress_load`, `environmental_burden`, `behavioral_instability`, `missingness_pressure`. Each axis is z-scored at the cohort level.

**Version.** 0.1 (research prototype). **Owner.** Research group running this repository. **Contact.** See `pyproject.toml`. **License.** MIT with an additional non-clinical / non-medical-device clause; see `LICENSE`.

## Intended use

**Primary use.** Research-grade modeling and simulation of human physiological and behavioral dynamics under environmental and behavioral perturbations. Suitable for dynamical-systems analysis of passive-sensing cohorts, pre-clinical mechanistic studies, methodological benchmarking against alternative state-space and time-series approaches, and counterfactual reasoning over simulated trajectories.

**Out-of-scope uses.** Any clinical, diagnostic, triage, prescriptive or treatment-recommendation use. Personalized health coaching delivered to end users. Any use that conveys probability of clinical events to a person or to a clinician acting on a clinical decision. Any use as a medical device under any jurisdiction's regulatory framework.

## Training data

The released code base is exercised on a fully synthetic cohort generated by `src/data/synthetic_generator.py` under a parameterized hierarchical model calibrated to plausible human ranges. No real participant data are included. The framework is structured to accept real cohorts via the adapter layer (`src/adapters/`), but no such cohorts are released here.

## Performance

On the synthetic cohort, one-step-ahead latent RMSE for the dynamics prior is reported in `results/tables/training_metrics.json`. The `results/model_leaderboard.csv` lists baseline performance for logistic regression, random forest, GRU, transformer, and the latent dynamics model on a downstream regime-classification task; calibration error, trajectory RMSE, and induced-missingness robustness are reported per model. These numbers describe behavior on the synthetic distribution. Real-cohort performance is unknown and is the central open question.

## Evaluation

Standard regression and classification metrics. Calibration via reliability curves and expected calibration error. Robustness under induced missingness up to 60%. Subgroup analyses across age, sex, climate-vulnerability tertiles, and heat-wave exposure. Out-of-distribution environmental shock tests at $1.25\times$, $1.5\times$, and $2\times$ baseline. Representation quality via PCA explained variance and cluster separation. No evaluation against any clinical outcome.

## Ethical considerations

Passive sensing data are highly identifying even when individual channels are coarse. The repository ships no real data and does not log any tool input. Climate-vulnerability subgroups overlap structurally with socioeconomic vulnerability, so the repo publishes disaggregated performance to make these gaps visible. Clinical and risk language is automatically sanitized at every output boundary, and the canonical disclaimer is attached to every API and dashboard response. Risk language is not surfaced directly to users. Downstream consumers should preserve these guardrails.

## Caveats and recommendations

This is a synthetic-data prototype. Generalization to real cohorts is not established.

Identifiability of the dynamics drift $f_\theta$ has not been established formally. See `paper/limitations.md`.

The perturbation operator is calibrated against plausible literature-derived effect sizes, not against participant-level causal estimates. Treat outputs as simulation, not as effect estimates.

Re-fit on your own cohort if you want to use this for anything. The synthetic-to-real similarity report (`src/evaluation/synthetic_to_real.py`) can be used to assess whether the transfer is even feasible.
